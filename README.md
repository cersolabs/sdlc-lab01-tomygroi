### Actividad 1
Teniendo en cuenta el material de clase, responda las siguientes preguntas:

1. ¿Por qué el desarrollo de software no puede realizarse simplemente comenzando a programar?

   R: Porque el desarrollo de software requiere analizar las necesidades del cliente, planificar el proyecto, diseñar la solución y seguir una serie de etapas ordenadas. Si se comienza directamente a programar pueden aparecer errores, requerimientos mal entendidos y problemas que afectan la calidad del software

2. ¿Qué significa que un desarrollo sea dirigido por un plan?

   R: Significa que antes de comenzar el desarrollo se analiza el problema y se crea un plan detallado con etapas, tiempos, recursos y requerimientos definidos. Ese plan guía todo el proceso de desarrollo hasta la finalización del sistema, para que este mejor organizado el desarrollo y mucho mas "formal"

3. ¿Cuáles son las ventajas de utilizar un plan de desarrollo?

   R: Permite estimar costos y tiempos, organizar las etapas del proyecto, hacer el desarrollo más predecible y asegurar que el software final cumpla con los requerimientos definidos al inicio

4. ¿Qué críticas se hacen a los modelos tradicionales de desarrollo?

   R: Se critica que son poco flexibles, permiten pocos cambios una vez iniciado el proyecto, requieren definir todos los requerimientos al comienzo, hay poca interacción con el cliente y los errores de etapas iniciales pueden arrastrarse hasta el final del proyecto

5. ¿Por qué en la práctica muchas organizaciones combinan metodologías ágiles y modelos dirigidos por un plan?

   R: Porque cada enfoque tiene ventajas diferentes. Los modelos dirigidos por plan permiten organización y planificación, mientras que las metodologías ágiles permiten adaptarse a cambios. Por eso se combinan prácticas, para lograr mejores resultados

| Modelo      | Característica principal | Cuándo conviene usarlo |
| ----------- | ------------------------ | ---------------------- |
| Cascada     |Etapas secuenciales donde cada fase se completa antes de pasar a la siguiente|Proyectos pequeños o con requerimientos bien definidos|

| RAD         |Desarrollo rápido usando herramientas automáticas|Cuando se necesita desarrollar software en poco tiempo|
